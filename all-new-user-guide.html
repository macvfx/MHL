Mini Guide • New Users • Media Trust Tools

Copy, verify, and prove it.

This one-page guide introduces the three tools new operators need most: CopyTrust for managed ingest, Drop Verify for one-folder artifact creation, and Folder Copy Compare for a direct “did the copy work?” answer.

CopyTrust = ingest workflow
Drop Verify = trust package
Folder Copy Compare = post-copy check

Start with this rule of thumb

  • Use CopyTrust when you are copying from a source to one or more destinations.
  • Use Drop Verify when the folder is already copied and you want MHL, PDF, and CSV artifacts.
  • Use Folder Copy Compare when you simply need to compare two folders and repair gaps.

Choose The Right Tool

These apps overlap on purpose, but each one is optimized for a different operator question.

CopyTrust

I need to offload media safely.

Best for camera cards, staged copy sessions, relay chains, and verified multi-destination ingest.

  • One source to one or more destinations
  • Queued jobs and relay chains
  • Copy, verify, MHL, receipts, logs
Drop Verify

I already have a folder and need proof.

Best for creating a deliverable trust package from a single folder without running a copy session.

  • Creates .mhl, contact sheet PDF, EXIF/media CSV
  • Reads one dropped folder recursively
  • Great for handoff, reporting, archive prep
Folder Copy Compare

I want to compare two folders.

Best for confirming a copy made by CopyTrust, Finder, P5, rsync, or another tool.

  • Quick Scan or Full Scan
  • Shows missing, extra, different, identical
  • Can copy missing files and generate or verify MHL

Fast Decision Flow

A visual shortcut for new users who just need to know where to begin.

Quick Reading

Three simple operator questions

  1. Am I copying right now? Use CopyTrust.
  2. Do I already have the folder and need proof artifacts? Use Drop Verify.
  3. Do I just need to compare two folders? Use Folder Copy Compare.
New users usually do best when they ignore advanced options until the first successful run is complete.

CopyTrust Quick Start

The in-app help is designed to keep the first job simple: add one source, add one destination, confirm preflight, and start the session.

Quick Start

First safe copy

  1. Add one source and one destination.
  2. Check that preflight looks clean.
  3. Decide whether this session should create a contact sheet PDF, EXIF/media CSV, or both.
  4. If you need richer metadata, confirm ExifTool is enabled.
  5. If PDF is on, confirm ffmpeg for MXF and REDline for R3D when needed.
  6. Click Start This Session.
During the run, Review & Verify lets you inspect the session without ending it.

CopyTrust Workflow Options

New users should start with the direct session, then graduate to relay chains or mixed queues only when the job truly needs them.

Method 1

Direct Multi-Destination Copy

Use for A -> B or A -> B and A -> C.

  • Add one source.
  • Add one or more destinations.
  • Click Start This Session.
  • Review with Review & Verify, then end with Review Summary….
Method 2

Relay Chain

Use for A -> B -> C when the first destination is the fast landing point.

  • Add one source and ordered destinations.
  • Click Queue Relay Chain.
  • Click Start Queue.
  • Each verified stop becomes the source for the next stop.
Method 3

Mixed Queued Sessions

Use when different cards need different destination sets.

  • Build a setup.
  • Click Queue Current Session.
  • Build the next setup and queue it too.
  • Run everything with Start Queue.
Advanced Start

When to use relay copy

  1. Destination B is your fastest safe landing point.
  2. Destination C is slower, remote, or network-based.
  3. You want the original card free sooner.
  4. You want CopyTrust to stage the legs in a clear order with Queue Relay Chain.
Current guidance is to build relay copy from one source + ordered destinations, then queue it as one visible chain.

Drop Verify In One Screen

Drop Verify is the simplest way to create a trust package from a folder that already exists.

Drop Verify

What happens

  1. Open Drop Verify.
  2. Review settings and choose outputs: MHL, Contact Sheet PDF, EXIF Camera Metadata CSV.
  3. Drop a folder into the window or choose it manually.
  4. Drop Verify scans media recursively, hashes files, extracts metadata, writes CSV, renders the PDF, and saves artifacts.
  5. Artifacts are written into Drop Verify_Receipts/ in the folder, or to an export folder when needed.
If the source is read-only, Drop Verify can prompt for an export folder instead of trying to write to the original media.

Folder Copy Compare In One Screen

The original sanity-check tool in the suite: drop two folders, compare them, then repair or prove the result.

Folder Copy Compare

Best use cases

  1. Confirm a copy after Finder, P5, CopyTrust, rsync, or another tool.
  2. Choose Quick Scan for a fast surface check.
  3. Choose Full Scan for content-level proof with hashing.
  4. Use Copy All Missing to repair obvious gaps.
  5. Use the MHL menu to generate or verify a manifest after a full xxHash64 scan.
If your main question is “what differs between these two folders?”, this is usually the cleanest tool in the suite.
